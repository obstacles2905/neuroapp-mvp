# План разработки мобильного приложения (neuro-app)

Кросс-платформа: **iOS + Android**. Стек: **Expo + React Native + TypeScript**.  
Бэкенд: существующий **NestJS** в репозитории (`../backend`).

Документ задаёт **порядок работ** и **границы MVP**; детализация задач ведётся в трекере / отдельных issues.

---

## Цели MVP (кратко)

| Область | Содержание |
|--------|------------|
| Аккаунт | Вход, регистрация (согласование контракта с API), сессия, обновление токенов |
| Онбординг | Выбор тем/категорий, важность 1–10 → передача на бэк для персонализации |
| Обучение | Каталог → уроки → прохождение сценария урока |
| Мотивация | Прогресс, дни подряд (стрик), «как в Duolingo» (упрощённо на MVP) |
| Профиль | Недавние курсы/темы, стрик, базовая статистика |
| Срочно | «Тревожная кнопка» — короткий набор упражнений (центр навигации) |
| Микс | «Джем-сессия» — набор упражнений по темам пользователя |
| Задел | Подписка (архитектура / заглушка), **без** оплаты в первом релизе |
| Позже | Биометрия (пульс, камера, взгляд, голос, фото зрачка) — модуль, не в MVP |
| Позже | Сложные нативные сенсоры — итерации после стабилизации ядра |

### Фаза 1 — авторизация и профиль `me`

- [x] Экраны **логин / регистрация** (валидация, ошибки, состояния загрузки) — `app/(auth)/login`, `register`

**Выход:** пользователь стабильно входит и видит себя.

**Выход:** новый пользователь проходит ввод и попадает в каталог/дом.

### Фаза 4 — трекер прогресса и стрик

- [x] **Локальное + серверное** согласование: «день засчитан» при **завершённом** уроке (PATCH `completed`); начатый и брошенный — не стрик
- [x] **Стрик** (UTC): счётчик + дата в `/me`; первый шаг — завершение урока; **сброс стрика** для тестов (`POST /api/app/activity/streak/reset`)
- [x] **Календарь** в **профиле** (сегменты активных дней, навигация по месяцу, в общем стиле приложения); **API** `GET /api/app/activity/calendar?year&month` (даты `lesson_completed_at` в UTC)
- [ ] **Счётчик** на главной табе (опционально), джем/health check — отдельные события позже
- [x] **Сервер** как source of truth для стрика: поля в `app_users`

**Выход:** пользователь видит дни подряд и не теряет их при смене устройства (при наличии API).

## Биометрия и сенсоры (после ядра)

Не входит в MVP, но **архитектурно** закладывается сразу:

- Отдельный **feature-модуль** (например `features/biometrics/`), не смешивать с сценариями урока
- **Контракт**: `MeasurementSession` — тип сессии, вложения, фичи/результаты, статус, ошибки
- **Провайдеры** 8.1–8.5 как **подключаемые реализации** (пульс, камера, eye tracking, openSMILE/голос, фото зрачка)
- **Privacy**: согласия, минимизация хранения, шифрование/ретеншн — отдельный чеклист перед продом

План по био обновляется, когда зафиксированы **регуляторные** и **технические** ограничения по платформам.

## Критерии готовности MVP

1. Пользователь **регистрируется / входит**, проходит **онбординг**
2. Видит **релевантный каталог** и **проходит уроки**
3. Видит **прогресс и стрик**
4. Использует **тревожную кнопку** и **джем**
5. **Профиль** отражает активность; подписка **не блокирует** базовый путь
6. Биометрия **отсутствует** в пользовательском пути либо спрятана за флагом
