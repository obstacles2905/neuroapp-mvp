# Описание функционала админ-панели (neuro-app)

Веб-приложение: **Next.js 16** (App Router), порт разработки **`3001`**.  
Бэкенд: **NestJS** в репозитории (`../backend`), префикс Admin API: **`/api/admin`** (базовый URL задаётся переменной **`NEXT_PUBLIC_API_BASE`**, по умолчанию `http://127.0.0.1:3000/api/admin`).

Документ описывает **уже реализованный** функционал UI и server actions; детализация изменений API — в трекере / PR.

---

## Технологии (кратко)

| Область | Содержание |
|--------|------------|
| UI | React 19, Tailwind CSS 4, базовые компоненты (shadcn-паттерн), `lucide-react` |
| Состояние в конструкторе | Zustand (`lesson-builder-store`) |
| Drag-and-drop | `@dnd-kit` (сортировка слайдов в блоке) |
| Запросы к API | Server Components + `fetch` через `lib/api/server-client.ts` (cookie `admin_token`), Server Actions для мутаций |

---

## Аутентификация и доступ

| Возможность | Описание |
|-------------|----------|
| **Вход** | Страница `/login`: email + пароль → `POST .../auth/login`, JWT в **httpOnly** cookie `admin_token` (7 дней), редирект на `/dashboard`; ошибка → `?error=1`. |
| **Выход** | Кнопка в сайдбаре → удаление cookie, редирект на `/login`. |
| **Заявка на доступ** | Страница `/join`: закрытая модель — `POST .../auth/join-request` (email, пароль, опционально имя и комментарий); успех → `?sent=1`, ошибка → `?error=1`. |
| **Защита маршрутов** | `middleware.ts`: публичные только `/`, `/login`, `/join`; остальное без `admin_token` → редирект на `/login` с `next=`. |
| **Текущий пользователь** | `getMe()` в layout админки → отображение email в сайдбаре; роль используется для условного пункта меню. |

---

## Навигация и оболочка

| Маршрут | Назначение |
|---------|------------|
| `/` | Редирект на `/dashboard`. |
| `/dashboard` | Стартовая страница с карточками-ссылками на контент и пользователей. |
| `/content` | Content Builder — список категорий. |
| `/content/new` | Создание категории. |
| `/content/lessons` | Список уроков (опциональный query `?categoryId=`). |
| `/content/lessons/new` | Создание урока-черновика (`?categoryId=` предзаполняет категорию). |
| `/content/lessons/[lessonId]/builder` | Конструктор шагов урока (блоки и слайды). |
| `/users` | Список app-пользователей (аналитика). |
| `/users/[id]` | Детали пользователя и таблица прогресса по урокам. |
| `/settings/access-requests` | **Только `super_admin`**: заявки на доступ; иначе редирект на `/dashboard`. |

Сайдбар: **Dashboard**, **Content Builder**, **Users**; для **super_admin** дополнительно **«Заявки доступа»**.

## Content Builder — категории

| Возможность | Описание |
|-------------|----------|
| **Список** | `GET /categories`: карточки с названием/описанием (RU/UK/EN), статус Published/Draft, аудитория каталога (прод vs тест/флаг), теги, опционально ключ фичефлага. |
| **Создание** | Форма на `/content/new`: один активный язык (RU/UK/EN) для title/description; порядок; публикация; **каталог** — `production` или `experimental`; опционально **ключ фичефлага** для experimental. → `POST /categories`. |
| **Меню карточки** | Переименование (PATCH title), теги (PATCH tags), промоут в прод / демоут в тест (`promoteCategoryToProductionAction` / `demoteCategoryFromProductionAction`), удаление категории (только если не «активна» в смысле UI — см. ограничения в интерфейсе). |
| **Ссылки** | Переход к урокам категории, быстрый «Добавить урок». |

---

## Content Builder — уроки и конструктор

| Возможность | Описание |
|-------------|----------|
| **Список уроков** | `GET /lessons` или `GET /lessons?categoryId=…`: статус draft/published, ссылка в **конструктор шагов**, удаление черновика (см. ниже). |
| **Новый урок** | Выбор категории, локализованные названия (ru/uk/en), порядок в категории → `POST /lessons` со статусом **draft**. |
| **Удаление** | Доступно только для **draft** (опубликованный урок удалить через UI нельзя — см. подсказки в диалоге). |
| **Конструктор** | Загрузка `GET /lessons/:lessonId` с блоками и слайдами. Урок структурирован по **типам блоков** упражнения: `what_exercise`, `how_to_do`, `why_do_it`, `how_often`, `what_it_gives` (отображаемые названия и подсказки — в `lib/lesson-content-blocks.ts`). |
| **Слайды (шаги)** | На блок: добавление шага с выбором **типа** (`theory`, `animation`, `video`, `practice`, `biometrics`) → `POST .../blocks/:blockId/steps`; редактирование контента в панели; **перетаскивание порядка** → `PATCH .../steps/order`; удаление шага. |
| **Медиа** | Загрузка файла через `uploadLessonMediaAction` → `POST .../media/upload` (multipart), результат `s3Key` / URL для привязки в контенте слайдов. |
| **Публикация** | `publishLessonAction` → `POST /lessons/:lessonId/publish` с отображением ошибки от API при неудаче. |
| **Локали в редакторе** | Переключение вкладок локали для текстовых полей в шагах (компонент `locale-tab-bar` и смежная логика). |

Типы контента слайдов и структуры полей заданы в `lib/types/lesson-step.ts` (теория, видео/анимация с `s3_key`, практика, биометрия-заглушка и т.д.).

## Пользователи и аналитика (app users)

| Возможность | Описание |
|-------------|----------|
| **Список** | `GET /analytics/users`: таблица с отображаемым именем/email, счётчики завершённых и уроков в процессе, последняя активность, ссылка «Детали». |
| **Карточка пользователя** | `GET /analytics/users/:id`: прогресс по урокам — урок (локализованное название), статус, % завершения, активность. |

---

## Заявки на доступ (super_admin)

| Возможность | Описание |
|-------------|----------|
| **Список** | `GET /access-requests` — pending-заявки с email, именем, комментарием, датой. |
| **Решение** | Кнопки одобрения / отклонения → `POST /access-requests/:id/approve` или `.../reject` (одобрение создаёт аккаунт content editor с паролем из заявки — как в подсказке на странице). |

---

## Роли администратора (UI)

| Роль | Поведение в интерфейсе |
|------|-------------------------|
| **super_admin** | В сайдбаре пункт «Заявки доступа», доступ к `/settings/access-requests`. |
| Прочие роли с валидным JWT | Полный доступ к Dashboard, Content Builder, Users без раздела заявок. |

Точная матрица прав на бэкенде может дополнять это описание.

---

## Зависимость от бэкенда

Админка **не работает автономно**: при остановленном API (типично `ECONNREFUSED` на порту **`3000`**) загрузка защищённых страниц и server-side `fetch` к Admin API завершится ошибкой. Для локальной разработки нужен запущенный backend или корректный `NEXT_PUBLIC_API_BASE`.

---

## Известные особенности (на момент документа)

- Корень Next/Turbopack в монорепо может выдавать предупреждение о нескольких lockfile — см. `next.config`/доку Next.
- Конвенция `middleware` в Next 16 может сопровождаться предупреждением о переходе на «proxy» — файл пока именуется `middleware.ts`.
